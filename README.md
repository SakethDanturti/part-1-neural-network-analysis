dataset link - https://drive.google.com/drive/folders/1akV6po4Nrgkc3yQrJkzA6cJlV-wBvUYs?usp=sharing

1. ApproachThe objective of this project is to build a binary classification pipeline to predict customer churn using data-driven indicators. A deep Feed-Forward Neural Network (Artificial Neural Network) was designed using TensorFlow/Keras to capture non-linear interactions between customer demographics, account information, and behavioral patterns.
2. Implementation StepsData Cleaning & Handling: Irrelevant structural tracking columns (such as customer_id) were removed to prevent geometric noise. Missing parameters in numeric rows were handled safely.Feature Engineering & Preprocessing: * Continuous features (e.g., tenure_months, monthly_charges_inr) were scaled uniformly using MinMaxScaler to keep ranges between 0 and 1.Categorical text features (e.g., plan_type, payment_method) were transformed into standard numeric matrices via One-Hot Encoding.Data Splitting: The processed dataset was divided into an 80% training set and a 20% testing set, maintaining stratified balance across target classifications.Model Design: Built a Sequential multi-layer network with hidden layers configured with ReLU/Tanh activations, a Dropout layer (0.2) to combat potential overfitting, and a Sigmoid output layer to output continuous probability scores.Hyperparameter Tuning: Conducted three distinct empirical experiments adjusting structural capacity (hidden layers, units), learning rates, batch sizes, and activation functions to determine the optimal generalized layout.
3. Results SummaryBest Model Metrics: Optimized via a 2-layer tanh network running at a batch size of 64, yielding a final stable Test Loss of 0.0766.Test Accuracy: Consistently stabilized at 98.25% across baseline configurations.Key Finding (The Accuracy Paradox): Detailed examination using a Confusion Matrix revealed that the extreme class imbalance in the source dataset (98.45% non-churn vs. 1.55% churn) led the network to optimize its loss by cleanly predicting a blanket "No Churn" state for all testing variables.
